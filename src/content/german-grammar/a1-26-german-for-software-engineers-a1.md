## Ziel dieses Kapitels

In Tech-Teams in Deutschland brauchst du **einfache** aber **klare** Sätze: Standup, Tickets, Bugs, Deployments — auf A1-Niveau.

Am Ende kannst du:

- ein **Daily Standup** auf Deutsch strukturieren
- **Bugs und Tickets** einfach beschreiben
- **Deployment-Status** melden
- **technische Bitten** höflich formulieren

---

# 1. Tech-Vokabeln auf A1

| Deutsch | Englisch |
| --- | --- |
| der Bug / der Fehler | bug |
| das Ticket | ticket |
| der Server | server |
| die Anwendung / die App | application |
| das Deployment | deployment |
| der Build | build |
| der Code | code |
| die Datei | file |
| das Log | log |
| die API | API |

Viele Begriffe sind **Lehnwörter** — oft neutral: **das** Deployment, **der** Bug.

---

# 2. Daily Standup — einfache Struktur

> **Guten Morgen!**
> **Gestern habe ich** am Ticket #42 **gearbeitet**.
> **Heute werde ich** den Bug **fixen**.
> **Ich habe** ein **Problem** mit der API.
> **Ich brauche** Hilfe vom Backend-Team.

A1-Struktur: **Zeit + Verb + Objekt**

> **Heute** teste **ich** die Anwendung.
> **Gestern** war **ich** im Meeting.

---

# 3. Bugs beschreiben

> **Ich habe** einen Bug **gefunden**.
> Die Anwendung **startet nicht**.
> Der Server **antwortet nicht**.
> Es gibt einen **Fehler** im Log.
> **Ich kann** den Bug **nicht reproduzieren**.

Grammatik:

> **Das** System **ist** offline. (sein)
> **Ich habe** einen Fehler **gefunden**. (haben + Akkusativ)
> **Der** Build **schlägt fehl**. (feste Verbindung)

---

# 4. Tickets

> **Ich erstelle** ein Ticket.
> **Das** Ticket ist **offen** / **geschlossen**.
> **Kannst du** das Ticket **prüfen**?
> **Wir müssen** das Ticket **priorisieren**.
> **Der** Kunde **meldet** einen Fehler.

---

# 5. Deployments und Builds

> **Wir deployen** heute **um** 18 Uhr.
> **Der** Build **läuft** gerade.
> **Der** Build **ist** grün. / **Der** Build **ist** rot.
> **Das** Deployment **war** erfolgreich.
> **Wir müssen** **rollback** machen. (Denglisch im Team üblich)

---

# 6. Bitten und Zusammenarbeit

> **Kannst du** den Code **reviewen**?
> **Bitte schick** mir den Link.
> **Können Sie** mir **Zugang** geben?
> **Ich warte** auf dein Feedback.
> **Wir besprechen** das **im** Meeting.

Höflich im formellen Team:

> **Könnten Sie** mir den Log **schicken**? (A2)

---

# 7. Status-Meldungen (Chat)

> **Bin** im Meeting.
> **Bin** gleich zurück.
> **Deploy** startet jetzt.
> **Alles** läuft wieder.
> **Brauche** Hilfe — **kann** jemand **schauen**?

Kurzformen sind im Tech-Chat **normal** — in E-Mails an Kunden: volle Sätze.

---

# 8. Häufige Fehler

## Fehler 1 — der/das bei Lehnwörtern

> **das** Problem, **der** Server, **die** API (feminin, wie *die* Schnittstelle)

## Fehler 2 — Ich bin deployen

Falsch:

> Ich bin deployen.

Richtig:

> **Ich deploye** heute. / **Wir deployen** heute.

## Fehler 3 — Zu informell zum Kunden

Intern: *Der Build ist rot.*
Kunde: **Das Update** war **leider nicht** erfolgreich.

---

# 9. Mini-Standup (Beispiel)

> **A:** Guten Morgen! **Gestern habe ich** Ticket 128 **abgeschlossen**. **Heute teste ich** die neue API. **Blocker:** Ich **habe keinen** Zugang zum Staging-Server.
> **B:** **Ich schicke** dir den Zugang **nach** dem Standup.

---

# 10. Übungen

## Aufgabe 1 — haben oder sein?

Ergänze: Der Server ______ offline.

Lösung:

> **ist** offline.

## Aufgabe 2 — Bug melden

Schreibe einen Satz: I found a bug.

Lösung:

> **Ich habe** einen Bug **gefunden**.

## Aufgabe 3 — Modalverb

Ergänze: Wir ______ heute deployen. (müssen)

Lösung:

> **müssen** heute deployen.

## Aufgabe 4 — Zeit

Der Build startet ______ 20 Uhr.

Lösung:

> **um** 20 Uhr

## Aufgabe 5 — Verneinung

Die App startet nicht. — mit kein:

Lösung:

> Die App hat **keinen** Fehler im Code? — besser: **Die App startet nicht.** / Es gibt **keine** Verbindung.

## Aufgabe 6 — Höfliche Bitte

Bitte um Code-Review (du):

Lösung:

> **Kannst du** bitte meinen Code **reviewen**?

## Aufgabe 7 — Standup

Schreibe **drei Sätze**: gestern, heute, Blocker.

## Aufgabe 8 — Chat-Nachricht

Schreibe eine kurze Statusmeldung: Deployment erfolgreich.

Lösung:

> **Deployment** war **erfolgreich**. **Alles** läuft wieder.
